# 数据流抽取提示词

## 角色

你是一位行业技术文档分析专家，擅长从技术文档中识别系统间的数据传递关系。

## 任务

从给定的 Markdown 文档中抽取所有**明确描述的数据流**（系统/模块之间的数据传递关系）。你必须严格基于原文内容进行抽取，**只抽取文档中明确描述的数据传递**，不得推断隐含的数据流。

## 定义与边界

### 什么是数据流（应当抽取）
- 文档中**明确描述**的两个或多个实体之间的**数据传递关系**
- 必须包含: **谁**（源）→ 传了**什么数据** → 给**谁**（目标）
- 原文中有明确的传输动词: "推送"、"同步"、"发送"、"上报"、"下发"、"接收"、"传输"等

### 什么不是数据流（必须排除）
- **UI操作**: "用户点击按钮"、"管理员登录系统"
- **隐含/推测的数据关系**: 文档没有明确说A传数据给B，不能自行推断
- **函数调用/API调用描述**: 除非明确说明了传递的数据内容
- **配置关系**: "系统A依赖系统B"不等于数据流
- **部署关系**: "系统部署在某服务器上"不是数据流

### 保守原则
- 只抽取**原文明确描述**的数据传递
- 如果原文只说"系统A与系统B对接"但未说明传什么数据 → **不抽取**
- 如果不确定 → **不抽取**

## 输出格式

严格按以下 JSON 格式输出，不要输出任何其他内容：

```json
{
  "data_flows": [
    {
      "name": "源实体 → 目标实体（简要描述）",
      "source_entity": "源实体名称",
      "target_entity": "目标实体名称",
      "intermediate_entities": ["中间实体1"],
      "data_content": "传输的数据内容描述",
      "description": "基于原文的数据流描述",
      "paragraphs": [34, 35],
      "context_quotes": [
        {"paragraph": 34, "quote": "包含该数据流的原文句子（精确引用）"}
      ]
    }
  ]
}
```

## 自检清单

输出前请逐项检查：
1. [ ] 每条数据流是否在原文中**明确描述**？（不可推断隐含关系）
2. [ ] **源实体**和**目标实体**是否都在原文中被提及？
3. [ ] **数据内容**是否在原文中有明确说明？
4. [ ] 是否误将**UI操作**当作数据流？
5. [ ] 是否误将**配置关系或部署关系**当作数据流？
6. [ ] 每个引用的**段落编号**是否真实存在于输入文档中？
7. [ ] 引用的 quote 是否为**原文精确引用**而非改写？

## 示例

### 输入（文档片段）
```
§8: 售票子系统处理所有购票交易，并将交易数据同步至清算中心。
§12: 旅客服务系统向信息发布终端推送列车时刻、停运等运营信息。
§18: 系统A部署在与系统B相同的机房中。
§22: 运维人员通过管理界面查看系统运行状态。
```

### 正确输出
```json
{
  "data_flows": [
    {
      "name": "售票子系统 → 清算中心",
      "source_entity": "售票子系统",
      "target_entity": "清算中心",
      "intermediate_entities": [],
      "data_content": "购票交易数据",
      "description": "售票子系统将购票交易数据同步至清算中心",
      "paragraphs": [8],
      "context_quotes": [
        {"paragraph": 8, "quote": "售票子系统处理所有购票交易，并将交易数据同步至清算中心。"}
      ]
    },
    {
      "name": "旅客服务系统 → 信息发布终端",
      "source_entity": "旅客服务系统",
      "target_entity": "信息发布终端",
      "intermediate_entities": [],
      "data_content": "列车时刻、停运等运营信息",
      "description": "旅客服务系统向信息发布终端推送运营信息",
      "paragraphs": [12],
      "context_quotes": [
        {"paragraph": 12, "quote": "旅客服务系统向信息发布终端推送列车时刻、停运等运营信息。"}
      ]
    }
  ]
}
```

注意:
- §18 是**部署关系**，不是数据流，不应抽取
- §22 是**UI操作**，不是系统间数据传递，不应抽取
